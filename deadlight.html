OVERVIEW


Deadlight is a 2.5D sidescroller platform game developed for Xbox 360 and PC by Tequila Works. It was developed using Unreal Engine 3. I had been working on this game as lead programmer for nearly two years and the main part of the game was developed by only two programmers, near the end, two more programmers joined the team.



GAMEPLAY & AI


The camera system uses volumes to determine where the player is and how the camera has to act. Every camera volume is configured with some information such as distance camera to the player, if the camera has to follow the player, if the camera has to collide with the volume borders, etc. In every volume, you can define information for any of the four sides of the volume (top, bottom, left and right) so the camera can react differently if the player enters by the right or left.


The AI uses a typical machine state with three kinds of senses to "feel" the world around (sight, hearing and contact).


CUSTOM TOOLS


Although we used Unreal Engine, was necessary to create custom tools for our needs, we used C# to created them. One was a simple tool to generate Unreal localization files from a Microsoft Excel file. Another tool was a one-button-build-system (see image below).

You can select individual actions by pressing the specific button or you can press the start button and do all actions one behind the other. The performed actions are:


  • Update of the project from the Subversion server.
  • Build Unreal Engine and scripts.
  • Cook all necessary data from specific maps.
  • Generate Xbox 360 final file.
  • Run some TCR tests.
  • Package all data required by the publisher (Xbox executable, documentation, artwork, etc) in a zip file ready to be sent.
